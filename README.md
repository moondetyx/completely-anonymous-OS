# completely-anonymous-OS
I have an idea to create a completely anonymous, as much as possible, Linux distribution based on Arch, I can give ideas, but I need someone who can do all this, I have a long list, believe me, I need a specialist who can optimize all of this as well. Add your ideas if you want.

Here's a list, it's not complete yet, but I'll be adding more and more ideas:
1. Every single session is launched in a virtual machine and is not saved.
2. Write and use a script like VSS, Only for Arch, which will create a shadow copy of the snapshot, this will provide more security
3. Encryption of each stream
4. TMP service that will be which will check every chain at the hardware virtualization level and will not allow any errors at all
5. Vlan connection that is different for each virtual session, configure a firewall to prevent cross-traffic
6. Combo of symmetric and asymmetric encryption, even if someone wants to track your online activity
7. Write your own, separate browser, cut out everything from it as much as possible, namely the Java script and WebRTC, all telemetry also needs to be removed
8. MAC address spoofer
9. Monitoring of IPS/IDS, at any attempt unauthorized intrusion, deletes the session completely (or at least records logs about the intrusion, if deleting is too much)

For now, these are all the ideas, I will add more as they come in. Please suggest yours, I will be glad to hear from each of you, thanks in advance
